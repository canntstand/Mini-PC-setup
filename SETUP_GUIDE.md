# Гайд по настройке инфраструктуры: ServeHub-2
## ⚠️ Важно

1. **Командная строка:** Все команды выполняются строго из корня проекта.
2. **Поддерживаемые ОС:** Проект рассчитан на операционные системы, построенные на базе Linux (гарантирована работа на Ubuntu и Debian; на Arch Linux — только для локального сервера).
3. **Настройка DNS:** В DNS-панели регистратора (например, на Webnames) необходимо направить A-запись основного домена и wildcard-запись (`*`) на внутренний IP-адрес VPN — **`10.8.0.1`**. Это критически важно: весь трафик к вашим сервисам должен идти внутри защищенного туннеля AmneziaWG. Если направить домен на публичный IP сервера VPS, Nginx заблокирует внешние запросы из-за правила `deny all`.

## 1. Подготовка (на удаленном и локальном серверах)
1. **Обновление системы:** Обновите все пакеты в системе:
    - Для Ubuntu/Debian: `sudo apt update && sudo apt upgrade`
    - Для Arch Linux: `sudo pacman -Syu`
2. **Настройка SSH и пользователей:**
    - Создайте нового пользователя: `sudo adduser user` *(в Arch Linux используйте: `sudo useradd -m -G wheel user`)*.
    - Добавьте его в группу администраторов: `sudo usermod -aG sudo user` *(в Arch Linux пользователь уже будет находиться в группе wheel)*.
    - Добавьте ваш публичный SSH-ключ `ed25519` в файл `~/.ssh/authorized_keys` созданного пользователя.
    - Ограничьте доступ в конфигурационном файле `/etc/ssh/sshd_config` (или в файлах внутри папки `/etc/ssh/sshd_config.d/*.conf`), разрешив вход исключительно по ключам и переведя параметр `PasswordAuthentication` в состояние `no`.
    - Примените настройки и перезапустите службу: `sudo systemctl restart ssh`.
    - **ВАЖНО:** Не закрывая текущую сессию терминала, откройте новое окно и проверьте корректность входа по ключу командой: `ssh user@ip_сервера`.
    - Настройте защиту от брутфорс-атак: `chmod +x scripts/fail2ban_setup.sh && ./scripts/fail2ban_setup.sh`.
3. **Установка Docker:** Установите [Docker Engine](https://docs.docker.com/engine/install/).
4. **Установка AmneziaWG:** Установите [AmneziaWG Kernel Module](https://github.com/amnezia-vpn/amneziawg-linux-kernel-module) *(в Arch Linux: `yay -S amneziawg-dkms amneziawg-tools`)*.
5. **Запуск Docker:** Включите службу Docker и добавьте её в автозагрузку: `sudo systemctl enable --now docker`.
6. **Клонирование репозитория:** Установите Git и склонируйте проект: `git clone https://github.com/canntstand/ServeHub-2 && cd ServeHub-2`.
7. **Настройка окружения:** Создайте файл `.env` на основе примера `.env.example`. Файлы `.env` на локальном и удаленном серверах должны быть абсолютно идентичными.

## 2. На удаленном сервере (VPS)
1. **Инициализация окружения:** `chmod +x scripts/setup_remote.sh && sudo ./scripts/setup_remote.sh`
2. **SSH-туннелирование:** Для безопасного доступа к панели управления WireGuard пробросьте порт локально через SSH-туннель: `ssh -L 51821:127.0.0.1:51821 user@ПУБЛИЧНЫЙ_IP_VPS`
3. **Доступ к панели:** Перейдите в браузере по адресу `http://127.0.0.1:51821` для входа в веб-интерфейс управления туннелями `wg-easy`.
4. **Создание конфигураций WireGuard:**
    - **ВАЖНО:** Первым созданным клиентом в панели WireGuard должен быть именно **локальный (домашний) сервер**. Это гарантирует, что за ним закрепится внутренний IP-адрес `10.8.0.2`. На этот адрес Nginx на VPS жестко перенаправляет весь входящий трафик к локальным сервисам.
    - Скачайте полученный конфигурационный файл локального сервера и переименуйте его в `awg0.conf`.
    - После этого создайте конфигурации для остальных личных устройств (смартфона, ноутбука), которые последовательно получат IP-адреса `10.8.0.3` и далее.

## 3. На локальном сервере
1. **Перенос конфигурации VPN:** Перенесите скачанный файл `awg0.conf` с вашего рабочего ПК напрямую в целевую папку локального сервера `./configs/amnezia/awg0.conf`. (можно просто скопировать текст). *Убедитесь, что файл конфигурации WireGuard на локальном сервере находится строго по пути `./configs/amnezia/awg0.conf`, так как именно эту директорию монтирует контейнер `amnezia-client`.*
2. **Развертывание инфраструктуры:** `chmod +x scripts/setup_local.sh && sudo ./scripts/setup_local.sh`

## Пользование сервисами
1. **Вход на портал:** После успешного запуска перейдите по вашему домену на главную страницу (веб-портал), предварительно включив VPN-клиент AmneziaWG на вашем устройстве.
2. **Первичная настройка приложений:** Для сервисов Navidrome, Audiobookshelf, Vaultwarden и Portainer при первом открытии необходимо пройти быструю процедуру создания аккаунта администратора. Для доступа к системной панели администрирования Vaultwarden используется токен, заданный в переменной `SECRET_VAULTWARDEN_PASSWORD`.
3. **Автоматическое развертывание:** Сервисы Nextcloud, Grafana и другие настраиваются автоматически. Данные для первого входа в них соответствуют глобальным переменным `ADMIN_USER` и `ADMIN_PASSWORD` из вашего `.env`.
4. **Интеграция с Matrix:** Служебный аккаунт для Synapse (Matrix) создается автоматически. Скрипт деплоя направит вашей основной учетной записи инвайт в закрытую комнату. Примите его в клиенте Element, чтобы получать важные системные уведомления от Alertmanager.
5. **Рекомендованные приложения для повседневного использования:**
    - Облачное хранилище — *Nextcloud*
    - Менеджер паролей — *Bitwarden*
    - Мессенджер — *Element X*
    - Музыкальная коллекция — *Tempo*
    - Аудиокниги и подкасты — *Audiobookshelf*
    - VPN-клиент — *AmneziaWG*
6. **Прямое подключение по SSH внутри защищенного VPN-туннеля:**
    - К удаленному серверу (VPS): `ssh user@10.8.0.1`
    - К локальному серверу: `ssh user@10.8.0.2`
    - *(Обратите внимание: данные приватные IP-адреса будут доступны для подключения только тогда, когда на вашем текущем рабочем устройстве — ноутбуке или смартфоне — запущен и активен VPN-клиент AmneziaWG)*.
